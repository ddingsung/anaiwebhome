# PRD: ANAI 회사 소개 랜딩 페이지

## 1. 개요

### 1.1 프로젝트명
ANAI 회사 소개 랜딩 페이지

### 1.2 목적
제조 현장의 중소기업 대표 및 현장 관리자를 대상으로, ANAI가 제공하는 가치를 정서적·논리적으로 전달하는 원페이지 랜딩 사이트를 구축한다. 기술 중심이 아닌 **"현장의 불안과 피로를 해소해주는 파트너"**라는 포지셔닝을 시각적·언어적으로 일관되게 전달한다.

### 1.3 핵심 메시지
> "어제보다 오늘 더, 마음 놓이는 현장을 만듭니다."

### 1.4 타겟 사용자
| 구분 | 설명 |
|------|------|
| 1차 타겟 | 지방 중소 제조기업 대표(사장님) |
| 2차 타겟 | 현장 관리자, 공장장 |
| 공통 특성 | 기술 용어에 대한 거부감, 현장 본질(품질·숙련도)에 대한 자부심, 반복 업무·설비 고장·인력 부족에 대한 만성 불안 |

---

## 2. 디자인 원칙

### 2.1 정서적 접근 — '문제'가 아닌 '상태'를 건드린다
- 방문자가 가장 먼저 느끼는 감정은 **'불안함'**과 **'피로감'** (인력 부족, 설비 고장 우려, 불량 발생 등)
- "마음 놓이는 현장"이라는 표현으로 복잡한 설명 없이 정서적 보상을 먼저 제공
- '무엇을 하느냐'보다 **'어떤 상태로 만들어주느냐'**를 먼저 보여준다

### 2.2 가치 중심 선언 — '도구'가 아닌 '결과'를 약속한다
- "AI를 만듭니다"로 시작하면 사용자는 방어 기제를 갖게 됨
- "새로운 도구를 드리는 곳이 아닙니다"라고 선을 그어 심리적 허들을 낮춤
- "매듭을 풀고, 번거로움을 덜어낸다" → 제공하는 것은 **편리함이라는 결과물**

### 2.3 본질 존중 — 현장의 숙련도와 품질에 대한 리스펙트
- "현장의 본질적인 가치"라는 말로 현장의 숙련도와 품질을 존중
- 단순 외주 업체가 아닌 **현장의 가치를 이해하는 파트너**라는 인식 형성

### 2.4 보이지 않는 기술 — 공기처럼 작동하는 인프라
- AI/LLM 등 복잡한 기술이 전면에 드러나지 않도록 설계
- 사용자는 기술을 몰라도 됨. "물어보면 답이 나오네?", "고장 전에 알려주네?" 같은 **경험만 전달**

## 3. 페이지 구조 및 섹션 상세

### 3.1 섹션 구성 (스크롤 순서)

```
[Hero] → [해결사 큐레이션] → [가치 선언] → [서비스 상세] → [CTA/문의]
```

---

### Section 1: Hero (첫 화면)

**목적:** 정서적 공감 → 심리적 안도감 제공

| 항목 | 내용 |
|------|------|
| 메인 카피 | "어제보다 오늘 더, 마음 놓이는 현장을 만듭니다." |
| 서브 카피 | "저희는 단순히 새로운 도구를 드리는 곳이 아닙니다. 꼬여있던 공정의 매듭을 풀고, 반복되던 번거로움을 덜어내어 오직 현장의 본질적인 가치만 남게 합니다." |
| CTA 버튼 | "우리 현장도 가능할까요?" → 문의 섹션으로 스크롤 |
| 비주얼 | 깔끔하고 안정감 있는 제조 현장 이미지 또는 추상적 일러스트 (불안→안도 전환 느낌) |

**디자인 노트:**
- 텍스트 중심의 미니멀한 레이아웃
- 충분한 여백으로 안정감 연출
- 부드러운 페이드인 애니메이션으로 첫인상 강화

---

### Section 2: 해결사 큐레이션 (프로덕트 섹션)

**목적:** 메인 슬로건("번거로움을 덜어내겠다")을 구체적으로 분해하여 보여주기

4개의 카드 형태로 구성. 각 카드는 **감정적 키워드 → 변화 설명** 구조.

#### 01. 눈이 편해지는 검수
> "반복되는 확인 작업에 지치지 않도록, 맑고 정확한 시선을 현장에 더합니다. 사소한 실수도 용납하지 않는 꼼꼼함으로 품질을 지킵니다."

#### 02. 멈춤 없는 안심
> "숨어있는 위험의 신호를 미리 듣고 알려드립니다. 예기치 못한 중단 없이, 현장의 맥박이 일정하게 뛸 수 있도록 뒤에서 묵묵히 살핍니다."

#### 03. 가벼워지는 발걸음
> "동선을 깎아내고 기록을 자동화하여 현장의 무게를 덜어냅니다. 낭비되던 움직임이 사라진 자리에 효율적인 흐름만 흐르게 합니다."

#### 04. 대화하는 기록
> "두꺼운 서류 뭉치 대신 대화로 답을 얻으세요. 흩어져 있던 노하우를 한데 모아, 누구나 전문가처럼 막힘없이 일할 수 있게 돕습니다."

**디자인 노트:**
- 그리드 또는 수평 스크롤 카드 레이아웃
- 각 카드에 고유 아이콘/일러스트 (눈, 심장박동, 발자국, 말풍선 등)
- 스크롤 시 순차적으로 등장하는 인터랙션

### Section 3: 가치 선언

**목적:** ANAI의 철학과 접근 방식을 선언적으로 전달

4개의 가치 블록으로 구성:

#### 1) 시간의 재배치 — "반복의 시간은 덜어내고"
- 숙련공조차 하루 70% 이상을 단순 반복(확인, 기록, 이동)에 소비
- 기술 도입 = 기계를 들이는 것이 아니라 **'죽은 시간'을 삭제**하는 것
- 효과: 육안 검사 피로도, 재고 파악 동선, 수기 기록의 번거로움을 기술이 흡수

#### 2) 몰입 환경의 조성 — "본질의 가치는 더하고"
- 잡무가 사라진 자리에 생기는 것은 **'생각할 여유'**
- "왜 이 구간에서 오차가 생길까?" 같은 창의적 고민이 가능한 환경 리디자인
- 효과: 현장 인력은 '단순 작업자' → **'운영 전문가'**로 변화

#### 3) 소프트웨어 그 이상의 경험 — "단순한 소프트웨어를 만드는 게 아닙니다"
- 코드를 파는 개발사가 아닌 **현장의 문제를 해결하는 '해결사'** 선언
- "당신의 공장이 겪는 병목 현상을 해결하기 위해 기술을 수단으로 사용"
- 효과: 프로그램이 아닌 **든든한 파트너**라는 신뢰 형성

#### 4) 보이지 않는 인프라 — "보이지 않는 곳의 번거로움을 기술로 해결합니다"
- 좋은 서비스 디자인 = 사용자가 기술을 의식하지 않는 것
- AI/LLM은 공기처럼 배경에서 작동하며 현장을 부드럽게 만듦
- 효과: "물어보면 답이 나오네?", "고장 전에 알려주네?" 같은 편리한 경험만 전달

**디자인 노트:**
- 좌우 교차 레이아웃 또는 타임라인 형태
- 각 블록에 핵심 숫자나 키워드 강조 (예: "70%의 시간")
- 부드러운 스크롤 트리거 애니메이션

### Section 4: 서비스 상세

**목적:** 실제 제공하는 서비스를 구체적으로 설명 (신뢰 구축)

#### 01. AI 비전 품질 검수
**한 줄 요약:** "사람의 눈보다 꼼꼼하게, 지치지 않고 지켜봅니다."

| 항목 | 내용 |
|------|------|
| 해결하는 불편함 | 컨베이어 벨트 앞 하루 종일 육안 검사의 고단함. 숙련공마다 다른 판정 기준. 미세 결함 누락. |
| 변화 | 반복적 육안 검사 피로 제거 → 품질 데이터만 남김. 검수원은 불량 원인 분석에 집중하는 **운영 전문가**로 전환. |

#### 02. 설비 예지 보전
**한 줄 요약:** "기계가 멈추기 전에, 먼저 신호를 듣고 알려드립니다."

| 항목 | 내용 |
|------|------|
| 해결하는 불편함 | 갑작스러운 설비 고장 → 납기 지연 불안, 밤잠 설침. 소음/진동의 미세 변화 감지 불가. |
| 변화 | 예기치 못한 중단(Down-time) 제거 → 계획된 정비와 안정적 가동만 남음. 현장은 일정한 맥박으로 운영. |

#### 03. 동선 및 재고 최적화
**한 줄 요약:** "불필요한 한 걸음을 줄여, 현장의 흐름을 가볍게 만듭니다."

| 항목 | 내용 |
|------|------|
| 해결하는 불편함 | 물건 찾기, 재고 수량 확인을 위해 현장 헤매는 낭비 시간. 수동 기록의 오류. |
| 변화 | 작업자 동선 단축 → 집중력 향상. 물류 흐름 최적화로 현장 전체가 막힘없이 운영. |

#### 04. 지능형 문서 자동화 (LLM)
**한 줄 요약:** "서류를 뒤지는 대신, AI와 대화하며 노하우를 공유합니다."

| 항목 | 내용 |
|------|------|
| 해결하는 불편함 | 두꺼운 매뉴얼/현장 일지에서 정보 검색 시간 낭비. 신입 교육 시 반복 설명 부담. |
| 변화 | 기록이 살아있는 지식으로 전환. "어제 작업에서 주의할 점이 뭐야?"라고 물으면 AI가 즉시 요약. 누구나 숙련공처럼 업무 수행 가능. |

**디자인 노트:**
- 각 서비스별 독립적인 풀스크린 또는 반분할 레이아웃
- "불편함 → 변화" 구조를 시각적으로 Before/After 또는 전환 애니메이션으로 표현
- 서비스별 대표 비주얼(아이콘, 일러스트, 또는 목업 이미지)

---

### Section 5: CTA / 문의

**목적:** 전환(Conversion) — 방문자를 리드로 전환

| 항목 | 내용 |
|------|------|
| 헤드라인 | "우리 현장에도 적용할 수 있을까요?" |
| 서브 카피 | "현장의 이야기를 들려주세요. 가장 적합한 방법을 함께 찾아드리겠습니다." |
| 입력 필드 | 회사명, 담당자명, 연락처(전화번호), 간단한 현장 고민(선택) |
| CTA 버튼 | "상담 요청하기" |

**디자인 노트:**
- 따뜻하고 신뢰감 있는 톤의 폼 디자인
- 입력 필드는 최소한으로 유지 (진입 장벽 최소화)
- 제출 후 "접수되었습니다. 빠르게 연락드리겠습니다." 확인 메시지

## 4. 전체 스크롤 흐름 요약

```
┌─────────────────────────────────────────────────┐
│  Hero                                           │
│  "어제보다 오늘 더, 마음 놓이는 현장을 만듭니다"    │
│  정서적 공감 + 안도감 제공                         │
│  [CTA: 우리 현장도 가능할까요?]                    │
├─────────────────────────────────────────────────┤
│  해결사 큐레이션                                  │
│  4개 카드: 검수 / 안심 / 발걸음 / 기록              │
│  "구체적으로 어떤 번거로움을 덜어주는가"              │
├─────────────────────────────────────────────────┤
│  가치 선언                                       │
│  시간의 재배치 / 몰입 환경 / 그 이상의 경험 / 인프라  │
│  "왜 우리인가, 어떤 철학으로 접근하는가"              │
├─────────────────────────────────────────────────┤
│  서비스 상세                                      │
│  AI 비전 검수 / 설비 예지보전 / 동선·재고 / LLM     │
│  "실제로 무엇을 제공하는가" (신뢰 구축)              │
├─────────────────────────────────────────────────┤
│  CTA / 문의                                      │
│  "우리 현장에도 적용할 수 있을까요?"                 │
│  간단한 문의 폼                                   │
└─────────────────────────────────────────────────┘
```

## 6. 디자인 가이드라인

### 6.1 컬러 시스템
| 용도 | 색상 | 의미 |
|------|------|------|
| Primary | Deep Navy (#1B2A4A) | 신뢰, 안정, 전문성 |
| Accent | Warm Blue (#3B82F6) | 기술, 혁신 (과하지 않게) |
| Background | Soft White (#FAFBFC) | 깨끗함, 여유 |
| Text | Dark Gray (#1F2937) | 가독성 |
| Sub Text | Medium Gray (#6B7280) | 보조 설명 |
| Highlight | Soft Green (#10B981) | 긍정적 변화, 성장 |

### 6.2 타이포그래피
| 용도 | 크기 | 굵기 |
|------|------|------|
| 메인 카피 | 48-56px (데스크탑) / 28-32px (모바일) | Bold (700) |
| 서브 카피 | 18-20px (데스크탑) / 16px (모바일) | Regular (400) |
| 섹션 헤드라인 | 32-40px (데스크탑) / 24px (모바일) | SemiBold (600) |
| 본문 | 16-18px | Regular (400) |
| 캡션/라벨 | 14px | Medium (500) |

### 6.3 레이아웃
- 최대 너비: 1200px (컨텐츠 영역)
- 섹션 간 여백: 120-160px (데스크탑) / 80px (모바일)
- 충분한 여백을 통해 안정감과 여유 연출
- 모바일 퍼스트 반응형 설계

### 6.4 톤 앤 매너
- **따뜻하고 담백한 톤** — 과장 없이 진정성 있게
- **기술 용어 최소화** — AI, LLM 등은 서비스 상세에서만 최소한으로 사용
- **현장 존중의 언어** — "당신의 현장", "현장의 가치", "현장의 본질"
- **부드러운 인터랙션** — 급격한 전환 없이 자연스러운 스크롤 경험
